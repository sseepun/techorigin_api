# User API

This API set can be used by members, admins and super admins. To ensure that you are communicating with the API, you will need to include app_id (app.id in your environment file) in JSON Body for every request to the server.

# Read

Method Endpoint
GET API_URLapi/user/read

Body Variables

Parameter Type Description
app_id String App id

Response

Parameter Type Description
status Number Response status code
messages Array Response message array
data array on successful request
id String The unique identifier for this request
role_id String The unique identifier of role associated with user's permission
username String User username
firstname String User first name
lastname String User last name
is_password_set String
email String User email
profile String User profile
thai_id String User Thai id
thai_id_path String User Thai id URL
code String
facebook_id String User facebook id
google_id String User google id
last_ip String User last ip
status String User status
detail array on successful request
id String The unique identifier of user detail
user_id String The unique identifier of user associated with user detail
address String User address
phone String User phone number
title String User title
company String User company
company_address String User company address
company_phone String User company phone number
role array on successful request
id String The unique identifier of role
name String Role name
is_admin String Indicated whether role is admin
is_super_admin String Indicated whether role is super admin
is_default String
order String Indicated an authority of role
status String Role status

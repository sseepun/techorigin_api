# Admin API

This API set can be used by admins and super admins. To ensure that you are communicating with the API, you will need to include app_id (app.id in your environment file) in JSON Body for every request to the server.

# User Create

Method Endpoint
POST API_URLapi/admin/user-create

Body Variables

Parameter Type Description
app_id String App id
role_id String Role id
firstname String User first name
lastname String User last name
email String User email
username String User username
password String User password
password_confirm String User password confirmation
ip String Visitor IP address for action log

Response

Parameter Type Description
status Number Response status code
messages Array Response message array
data array on successful request
app_id String App id
firstname String User first name
lastname String User last name
email String User email
username String User username
password String User password
password_confirm String User password confirmation
ip String Visitor IP address for action log

Request Example


{
    "app_id": "5789541587749547",
    "role_id": 1,
    "firstname": "aaa",
    "lastname": "aaa",
    "email": "aaa@a.com",
    "username": "aaaaa",
    "password": "aaaaaa",
    "password_confirm": "aaaaaa",
    "ip": "127.127.127.2"
}

Response Example


{
    "status": 200,
    "messages": {
        "success": "สร้างข้อมูลสำเร็จ"
    },
    "data": {
        "app_id": "5789541587749547",
        "firstname": "aaa",
        "lastname": "aaa",
        "email": "aaa@a.com",
        "username": "aaaaa",

    }

Parameter Type Description

email String User email
profile String User profile
thai_id String User Thai id
thai_id_path String User Thai id URL

# User Update Password

Method Endpoint
POST API_URLapi/admin/user-update-password

Body Variables

Parameter Type Description
app_id String App id
id String User id
new_password String User new password
new_password_confirm String User new password confirmation

Response

Parameter Type Description
status Number Response status code
messages Array Response message array
data Boolean Return true on successful request

Request Example


{
    "app_id": "5789541587749547",
    "id": 2,
    "new_password": "aaaaaa",
    "new_password_confirm": "aaaaaa"
}

Response Example


{
    "status": 200,
    "messages": {
        "success": "แก้ไขข้อมูลสำเร็จ"
    },
    "data": true
}
